## üéØ Aanvullende examenkennis (DP-900)

### Module 1 ‚Äì Relationeel & SQL basics
- Tabellen = rijen (records) + kolommen (velden); **PK** uniek, **FK** verwijst naar PK in andere tabel.
- Normalisatie: 1NF (atomair), 2NF (volledig afhankelijk van PK), 3NF (geen transitieve afhankelijkheid); **Denormalisatie** voor performance/BI.
- SQL-categorie√´n: **DDL** (CREATE/ALTER/DROP), **DML** (SELECT/INSERT/UPDATE/DELETE), **DCL** (GRANT/REVOKE), **TCL** (BEGIN/COMMIT/ROLLBACK).
- Transacties: **ACID**; isolatieniveaus ‚Üí Read Committed vs Snapshot; locking vs row versioning.
- Querytypes: filter (WHERE), groeperen (GROUP BY/HAVING), joins (INNER/LEFT/RIGHT/FULL), aggregaties (SUM/COUNT/AVG), window-functies (ROW_NUMBER, RANK).

### Module 2 ‚Äì Azure SQL varianten & keuzes
- **Azure SQL Database (PaaS)**: single db, elastic pools, Hyperscale; automatisch patching/HA, autoscale; cloud-first.
- **Azure SQL Managed Instance (PaaS)**: bijna volledige SQL Server-compatibiliteit (SQL Agent, Service Broker, CLR), goed voor lift & shift.
- **SQL Server op VM (IaaS)**: volledige controle, zelf patch/HA regelen.
- Keuze: cloud-native ‚Üí SQL DB; migratie zonder wijzigingen ‚Üí Managed Instance; legacy features/OS-controle ‚Üí SQL VM.

### Module 2 ‚Äì Cosmos DB essentials
- **Partition key** bepaalt schaalbaarheid/kosten; kies hoge cardinaliteit, gelijk verdeelde access.
- **Throughput**: RU/s per container of database; autoscale vs provisioned; charge = RU per operatie.
- **Consistentielevels**: Strong > Bounded Staleness > Session (default) > Consistent Prefix > Eventual (sterk ‚Üí zwak).
- **API‚Äôs**: Core (SQL), Mongo, Cassandra, Gremlin (graph), Table.
- **Global distribution**: multi-region write/read, automatische failover; replicatieknoppen om regio toe te voegen.
- **Indexering** standaard aan; pas aan voor cost/perf (inclusief composites).

### Module 3 ‚Äì Security & netwerk
- Netwerk: **Firewall rules**, **Private Endpoints**, **VNet-integratie**; vermijd publieke exposure.
- Identiteit: **Entra ID** auth waar mogelijk; **MFA** voor beheer.
- **Auditing** + **Advanced Threat Protection** (SQL Defender) voor detectie.
- **CMK** vs Microsoft-managed keys; **TDE** at rest, **TLS** in transit; **Always Encrypted** client-side.
- **RLS** (Row-Level Security) en Dynamic Data Masking beperken data-exposure.

### Module 4 ‚Äì Analytics & modellen
- **Star-schema**: Feit-tabellen (measures, FK‚Äôs), Dimensies (context, PK‚Äôs), SCD type 1/2 basics.
- **ETL vs ELT**: ETL = transform v√≥√≥r opslag, ELT = transform in compute-engine (Synapse/Fabric).
- **Synapse SQL**: Dedicated pool (pre-provisioned, MPP) vs Serverless SQL (pay-per-query, ideaal voor lake).
- **Power BI**: Import vs DirectQuery vs Composite; VertiPaq kolom-gebaseerd; DAX voor measures.

### Module 5 ‚Äì Fabric detail
- **OneLake shortcuts** (ADLS/Delta) koppelen data virtueel zonder kopie.
- **Lakehouse** = files + managed tables; **Warehouse** = alleen SQL, BI-geoptimaliseerd.
- **Dataflows Gen2** voor no/low-code ingestie; **Notebooks** voor Spark-transformaties; **Pipelines** (Fabric) voor orkestratie.

### Data lifecycle & tools
- Ingest: Event Hubs (telemetry), IoT Hub (device mgmt + ingest), Service Bus (enterprise messaging), ADF/Fabric Pipelines (batch/ELT).
- Storage-keuze: Blob (object), ADLS Gen2 (big data, POSIX ACL), OneLake (Fabric).
- Monitoring/kosten: RU/s (Cosmos), DTU/vCore (SQL), DWU (Synapse Dedicated), Capacities (Fabric). Vraag ‚Äúwat drijft de kosten?‚Äù in scenario‚Äôs.

### Examenscenario‚Äôs om te herkennen
- Wereldwijde lage latency + multi-region writes ‚Üí Cosmos DB + juiste partition key + autoscale RU.
- Lift & shift SQL met minimale wijzigingen ‚Üí SQL Managed Instance (SQL Agent vereist).
- Query over CSV/Parquet in lake zonder ingest ‚Üí Synapse Serverless SQL of Fabric Lakehouse SQL endpoint.
- Real-time events naar BI ‚Üí Event Hub/IoT Hub ‚Üí Stream Analytics ‚Üí Power BI (push dataset).
- Self-service BI zonder infra ‚Üí Fabric (OneLake + Warehouse/Lakehouse + Power BI) met capacities.
- Governancevraag ‚Üí Purview (catalogus, classificatie, lineage) + sensitivity labels.

### Extra begrippen en examenhaakjes
- **Linked Service (ADF/Fabric Data Factory/Synapse Pipelines)**: connectiedefinitie naar een bron of sink (SQL, Blob, SaaS); wordt hergebruikt door **Datasets** en **Pipelines**; beveiligd met Key Vault of managed identity.
- **Integration Runtime (IR)**: compute + netwerk-koppeling voor kopie/activiteiten; Azure IR (PaaS), Self-hosted IR (on-prem/VNet), Managed VNet IR (beveiligd zonder zelf hosten).
- **Dataset / Dataflows**: beschrijven schema/locatie van data; **Dataflow** (Power BI/Fabric) voor no/low-code ingestie en transformatie.
- **Synapse engines**:
  - **Serverless SQL pool**: pay-per-query, leest Parquet/CSV uit lake (OPENROWSET, EXTERNAL TABLE); geen provisioning.
  - **Dedicated SQL pool**: geprovisioned MPP (DWU); geschikt voor grote warehousing, nodig voor performance/isolatie.
  - **Spark pool**: PySpark/Scala/SQL voor transformaties op lake (Delta/Parquet); notebooks.
  - **Data Explorer pool**: Kusto-engine voor time-series/events (KQL); snelle ingest/aggregaties.
- **Copy activity** vs **Data Flow** (ADF/Synapse): Copy = verplaatsen zonder zware transformatie; Data Flow = visuele mapping/transformatie (Spark) zonder code.
- **PolyBase / COPY INTO**: snelle bulkload naar dedicated SQL pool vanuit lake/Blob.
- **Result-set caching (Synapse Serverless)**: cachet queryresultaat voor dezelfde query en ongewijzigde bestanden; kan kosten verlagen.
- **Workload-isolatie** in dedicated SQL pool: via resource class/ workload management voor concurrerende queries.
- **Purview (Microsoft Purview)**: data catalogus + lineage + classificatie + toegangspolicies; scan bronnen (SQL, ADLS, Synapse, Power BI); labels en RBAC-integratie. Examenvraag ‚Üí governance, datacatalogus, lineage, gevoelige data markeren.
- **Kostenhoek**: Serverless = per TB gescand; Dedicated SQL = DWU vast; Cosmos = RU/s; Fabric = capacity-units; ADF = activiteit + IR runtime.
